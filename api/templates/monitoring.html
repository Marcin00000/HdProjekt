Czym jest drift danych?

Drift oznacza, że rozkład nowych danych znacząco różni się od danych, na których uczono model — co może obniżyć jakość prognoz.

Jak to działa?

  • Baseline to training_baseline.parquet — zapisany podczas ostatniego treningu
  • Bieżący silver (cleaned.parquet) jest porównywany z baseline przez Evidently
  • Raport Evidently analizuje rozkłady kolumn testem statystycznym
  • Jeśli odsetek dryfujących kolumn przekroczy próg, system inicjuje auto-retrain przez Prefect

Pasek driftu

Wskazówka na niebiesko to aktualny poziom driftu. Jeśli przekracza czerwoną linię progu, wyzwalany jest retrain.

Tryb demo

Brak pliku silver? Można uruchomić symulację driftu — system generuje sztucznie zmienione dane, aby przetestować mechanizm monitoringu.

Historia raportów

Każdy run zapisuje raport HTML Evidently w monitoring/reports/. Można go pobrać z listy historii.
