Wnioskowanie w Czasie Rzeczywistym (Real-time Inference)

Formularz prognozowania wykorzystuje produkcyjny model XGBoost do natychmiastowego oszacowania rocznej pensji brutto (w USD). Po kliknięciu przycisku "Przewiduj", parametry wejściowe są przekazywane do FastAPI, które ładuje preprocesor oraz model bezpośrednio z pamięci RAM serwera.

Preprocesor i Transformacja Danych

Wprowadzone wartości nie mogą trafić bezpośrednio do algorytmu XGBoost. Przechodzą one przez identyczny proces transformacji (Pipeline scikit-learn), jaki zastosowano podczas treningu:

  • One-Hot Encoding: Kolumny kategoryczne (stanowisko, wykształcenie, branża, rozmiar firmy, praca zdalna, lokalizacja) są zamieniane na wektory binarne (0/1).
  • Skalowanie i Inżynieria Cech: Zmienne numeryczne są odpowiednio sprawdzane, a flaga pracy zdalnej jest mapowana na binarny wskaźnik is_remote.

Walidacja Pydantic i REST API

Każde zapytanie podlega automatycznej walidacji za pomocą klasy Pydantic SalaryPredictRequest, co chroni serwer przed błędnymi typami danych (np. tekst w polu lat doświadczenia). Prognozy można pobierać programistycznie za pomocą REST API:

  • POST /predict — z JSON body (zwraca wyliczoną kwotę).
  • Dokumentacja Swagger: Swagger UI ↗.

Ograniczenia i Ostrzeżenia (Warnings)

  • Zasada interpolacji: Model uczy się na historycznych wzorcach. Podanie skrajnych wartości (np. 40 lat doświadczenia przy stanowisku juniorskim) może skutkować nietypowymi wynikami.
  • Detekcja nietypowych danych (Out of Distribution): Jeśli podane parametry rzadko występowały w zbiorze treningowym, system wyświetli ostrzeżenie informujące o potencjalnie wyższym błędzie prognozy.
