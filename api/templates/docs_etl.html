Pipeline medalionowy

Dane przechodzą przez trzy warstwy, każda z wyższą jakością:

  • Raw — oryginalny plik CSV (Kaggle), bez zmian
  • Silver — oczyszczone: usunięte duplikaty, outliery salary, puste job_title/location; dodane cechy salary_k, high_skills, is_remote
  • Gold — agregaty analityczne gotowe na dashboard (salary_by_location, salary_by_location_industry)
  • Azure SQL — schemat gwiazdy: fact_salaries, dim_job, dim_company, dim_time

Kiedy użyć każdego kroku

  • Krok 1 — Przygotowanie lokalne: pierwsza instalacja lub nowy plik CSV. Zapisuje silver + gold na dysku. Nie wymaga Azure.
  • + Eksportuj do Azure Lake: opcja do kroku 1 — wgrywa parquet do ADLS Gen2. Wymaga skonfigurowanego Azure Storage.
  • Krok 2 — Ładowanie hurtowni: po kroku 1, gdy skonfigurowane Azure SQL. Tworzy/nadpisuje tabele wymiarów i faktów.
  • Krok 3 — Pipeline pełny (Prefect): gdy wszystko skonfigurowane. Uruchamia kroki 1–2 przez orkiestrację Prefect (lub synchronicznie jako fallback).
  • Krok 3 bez SQL: jak wyżej, ale pomija załadowanie hurtowni. Użyteczny w środowisku Docker bez sterownika ODBC.

Wymagania — CSV

Plik musi zawierać kolumny:

  • job_title, experience_years, education_level, skills_count
  • industry, company_size, location, remote_work
  • certifications, salary

Wymagania — Azure

  • Storage (Lake): AZURE_STORAGE_ACCOUNT + AZURE_STORAGE_KEY lub AZURE_STORAGE_SAS_TOKEN
  • SQL (DWH): AZURE_SQL_SERVER, AZURE_SQL_DATABASE, AZURE_SQL_USERNAME, AZURE_SQL_PASSWORD

Pliki wyjściowe

  • data/processed/cleaned.parquet — warstwa silver
  • data/processed/salary_by_location.parquet — gold agregat 1
  • data/processed/salary_by_location_industry.parquet — gold agregat 2
  • data/processed/prepare_summary.json — podsumowanie kroku 1
  • data/processed/etl_summary.json — podsumowanie pełnego ETL
  • data/processed/dwh_summary.json — podsumowanie ładowania SQL

Schemat gwiazdy (Azure SQL)

  • fact_salaries — fakty: salary, klucze wymiarów
  • dim_job — stanowisko, wykształcenie, umiejętności, certyfikaty
  • dim_company — branża, wielkość firmy, praca zdalna
  • dim_time — data ETL (jeden wiersz per uruchomienie)
