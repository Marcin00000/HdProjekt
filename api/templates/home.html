O projekcie

System prognozowania wynagrodzeń w IT i na rynku pracy powstał jako kompleksowy projekt integrujący nowoczesne podejście do inżynierii danych (Data Engineering) oraz uczenia maszynowego (MLOps). Portal umożliwia automatyczne pobieranie, oczyszczanie, składowanie w chmurze Azure, analizę, predykcję oraz monitorowanie jakości modeli.

Cykl życia danych i potok (Pipeline)

  1. Przygotowanie i ETL: Automatyczny pobór danych źródłowych z formatu CSV, walidacja jakościowa w procesie ETL (warstwy Silver/Gold) oraz migracja i załadowanie danych do relacyjnej hurtowni danych Azure SQL Database w zoptymalizowanym schemacie gwiazdy.
  2. Uczenie i wersjonowanie (ML): Cykliczny trening modelu XGBoost z automatycznym strojeniem hiperparametrów (Grid Search) i walidacją krzyżową. Wszystkie metryki oraz artefakty są wersjonowane za pomocą narzędzia DVC (z backendem na Azure Blob Storage) i logowane w serwerze MLflow.
  3. Serwowanie i analiza: Udostępnienie interaktywnego, responsywnego dashboardu analitycznego z wykresami Chart.js oraz interfejsu webowego i API REST (FastAPI) do wnioskowania (predykcji) w czasie rzeczywistym.
  4. Monitoring i jakość (MLOps): Ciągły nadzór nad stabilnością środowiska poprzez wykrywanie dryfu danych (Data Drift) przy użyciu biblioteki Evidently. W przypadku wykrycia pogorszenia parametrów danych, orkiestrator Prefect automatycznie uruchamia procedurę ponownego uczenia (auto-retrain).

Struktura i technologie

  • Przechowywanie danych: Azure Data Lake Storage Gen2, Azure SQL Database.
  • Przetwarzanie i ML: Python (Pandas, Scikit-Learn, XGBoost), DVC (Data Version Control).
  • Śledzenie i MLOps: MLflow, Prefect, Evidently.
  • Backend i UI: FastAPI, Jinja2, Chart.js, HTML5, Vanilla CSS.
  • Infrastruktura: Konteneryzacja za pomocą Docker i Docker Compose.

Szybki start (Local Dev)

Skopiuj plik konfiguracji .env.example jako .env i dostosuj parametry bazodanowe oraz chmurowe. Umieść surowe dane CSV w folderze input/ i uruchom całe środowisko komendą docker compose up --build.
